# 프론트엔드 기초

## 🎨 프론트엔드란?

프론트엔드는 **사용자가 직접 보고 상호작용하는 부분**입니다. 버튼, 입력창, 이미지 등 브라우저에 표시되는 모든 것이 프론트엔드입니다.

## ⚛️ React란?

React는 **사용자 인터페이스를 만들기 위한 JavaScript 라이브러리**입니다.

### 왜 React를 쓸까?

일반 HTML/JavaScript로도 웹사이트를 만들 수 있지만, 복잡한 웹 애플리케이션을 만들 때는:
- 코드가 너무 길어지고 복잡해집니다
- 같은 코드를 여러 번 반복 작성해야 합니다
- 데이터가 바뀔 때 화면을 업데이트하기 어렵습니다

React는 이런 문제를 **컴포넌트**라는 개념으로 해결합니다!

## 🧩 컴포넌트 (Component)

컴포넌트는 **재사용 가능한 UI 조각**입니다. 레고 블록처럼 여러 개를 조립해서 완전한 페이지를 만듭니다.

### 바이브 프로젝트의 컴포넌트 예시

```jsx
// Header.jsx - 상단 네비게이션 바
function Header({ user, onLogout }) {
  return (
    <header className="header">
      <h1>Vibe</h1>
      <div>
        <span>{user.username}님</span>
        <button onClick={onLogout}>로그아웃</button>
      </div>
    </header>
  );
}
```

**이 코드의 의미:**
- `Header`라는 컴포넌트를 만듭니다
- `user`와 `onLogout`을 외부에서 받아옵니다 (이걸 **props**라고 함)
- JSX를 사용해서 HTML처럼 보이는 UI를 정의합니다
- 사용자 이름을 표시하고, 로그아웃 버튼을 만듭니다

## 📄 JSX (JavaScript XML)

JSX는 **JavaScript 안에서 HTML처럼 보이는 문법**을 쓸 수 있게 해줍니다.

```jsx
// 일반 HTML
<div class="container">
  <h1>제목</h1>
</div>

// JSX
<div className="container">
  <h1>제목</h1>
</div>
```

**차이점:**
- `class` 대신 `className` 사용 (class는 JavaScript 예약어라서)
- JavaScript 표현식을 `{}` 안에 넣을 수 있습니다

```jsx
const name = "철수";
return <h1>안녕하세요, {name}님!</h1>;  // "안녕하세요, 철수님!" 출력
```

## 🗂️ 바이브 프로젝트의 프론트엔드 구조

```
frontend/src/
├── App.jsx              # 최상위 컴포넌트, 전체 앱의 시작점
├── main.jsx             # React 앱을 DOM에 연결
├── components/          # 재사용 가능한 컴포넌트
│   └── Header.jsx       # 상단 네비게이션 바
├── pages/               # 각 페이지 컴포넌트
│   ├── Login.jsx        # 로그인 페이지
│   ├── PostList.jsx     # 게시물 목록 페이지
│   ├── PostDetail.jsx   # 게시물 상세 페이지
│   └── PostForm.jsx     # 게시물 작성/수정 페이지
└── services/
    └── api.js           # 백엔드와 통신하는 코드
```

## 📱 페이지 예시: 로그인 페이지

```jsx
// Login.jsx (단순화된 버전)
function Login({ onLogin }) {
  const [username, setUsername] = useState('');
  const [password, setPassword] = useState('');

  const handleSubmit = async (e) => {
    e.preventDefault();
    
    // 백엔드 API 호출
    const response = await authAPI.login(username, password);
    
    if (response.data.success) {
      onLogin(response.data.user);  // 로그인 성공!
    }
  };

  return (
    <div className="login-page">
      <form onSubmit={handleSubmit}>
        <input
          type="text"
          value={username}
          onChange={(e) => setUsername(e.target.value)}
          placeholder="사용자명"
        />
        <input
          type="password"
          value={password}
          onChange={(e) => setPassword(e.target.value)}
          placeholder="비밀번호"
        />
        <button type="submit">로그인</button>
      </form>
    </div>
  );
}
```

**이 코드가 하는 일:**

1. **상태 관리**: `useState`로 username과 password를 저장
2. **이벤트 처리**: 입력창에 타이핑할 때마다 상태 업데이트
3. **폼 제출**: 로그인 버튼을 누르면 `handleSubmit` 실행
4. **API 호출**: 백엔드에 username/password 전송
5. **성공 처리**: 로그인 성공하면 `onLogin` 호출

## 🎭 Props vs State

### Props (속성)
- **부모 컴포넌트가 자식에게 전달하는 데이터**
- 읽기 전용 (자식이 바꿀 수 없음)

```jsx
// 부모가 자식에게 데이터 전달
<Header user={currentUser} onLogout={handleLogout} />

// Header 컴포넌트에서 받아서 사용
function Header({ user, onLogout }) {
  return <span>{user.username}</span>;
}
```

### State (상태)
- **컴포넌트가 관리하는 동적 데이터**
- 변경 가능 (setState로 업데이트)

```jsx
const [count, setCount] = useState(0);  // 초기값 0

// 버튼 클릭하면 count 증가
<button onClick={() => setCount(count + 1)}>
  클릭 횟수: {count}
</button>
```

## 🔄 컴포넌트 생명주기와 useEffect

컴포넌트는 생성되고, 업데이트되고, 사라집니다. `useEffect`는 **특정 시점에 코드를 실행**하게 해줍니다.

```jsx
// App.jsx에서
useEffect(() => {
  checkAuth();  // 컴포넌트가 처음 화면에 나타날 때 실행
}, []);  // 빈 배열 = 한 번만 실행
```

**실제 사용 예시:**
```jsx
function PostList() {
  const [posts, setPosts] = useState([]);

  useEffect(() => {
    // 페이지 로드될 때 게시물 목록 가져오기
    const fetchPosts = async () => {
      const response = await postsAPI.getPosts();
      setPosts(response.data.posts);
    };
    
    fetchPosts();
  }, []);  // 페이지 로드 시 한 번만 실행

  return (
    <div>
      {posts.map(post => (
        <div key={post._id}>
          <h2>{post.title}</h2>
          <p>{post.content}</p>
        </div>
      ))}
    </div>
  );
}
```

## 🎯 핵심 정리

1. **React**: 컴포넌트 기반으로 UI를 만드는 라이브러리
2. **컴포넌트**: 재사용 가능한 UI 조각 (레고 블록 같은 것)
3. **JSX**: JavaScript 안에서 HTML처럼 쓸 수 있는 문법
4. **Props**: 부모가 자식에게 전달하는 데이터 (읽기 전용)
5. **State**: 컴포넌트가 관리하는 변경 가능한 데이터
6. **useEffect**: 특정 시점에 코드를 실행 (데이터 로드 등)

## 🤔 실제로 어떻게 작동할까?

**사용자가 게시물 목록을 볼 때:**

1. 사용자가 브라우저에서 바이브 사이트 접속
2. `App.jsx`가 실행됨
3. `useEffect`로 로그인 상태 확인
4. 로그인되어 있으면 `PostList` 컴포넌트 표시
5. `PostList`의 `useEffect`로 백엔드에서 게시물 데이터 가져옴
6. 데이터를 받으면 `setPosts`로 상태 업데이트
7. 화면에 게시물 목록이 표시됨
