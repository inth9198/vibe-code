# 시작하기 - 바이브 프로젝트 실행 방법

## 🎯 목표
바이브 프로젝트를 컴퓨터에서 실행시키기

## 📁 프로젝트 구조 (간단 버전)

```
vibe/
├── frontend/       # 화면 (사용자가 보는 부분)
│   └── src/
│       ├── pages/      # 각 페이지들
│       └── components/ # 재사용하는 UI 조각들
│
├── backend/        # 서버 (데이터 처리)
│   └── src/
│       ├── routes/     # API 주소들
│       └── models/     # 데이터베이스 구조
│
└── 강의자료/       # 지금 보고 있는 문서들
```

## 🚀 실행 방법

### 1단계: 프로그램 설치 (처음 한 번만)

필요한 프로그램:
- **Node.js** - JavaScript 실행 환경
- **MongoDB** - 데이터베이스 (또는 Docker 사용)
- **코드 에디터** - VS Code 추천

### 2단계: 프로젝트 설정

```bash
# 1. 프론트엔드 폴더로 이동
cd frontend

# 2. 필요한 패키지 설치
npm install

# 3. 백엔드 폴더로 이동
cd ../backend

# 4. 필요한 패키지 설치
npm install
```

### 3단계: 실행하기

**터미널 2개를 열어야 합니다!**

**터미널 1 - 백엔드 실행:**
```bash
cd backend
npm run dev
```
→ "서버가 포트 3000에서 실행 중입니다." 메시지가 뜨면 성공!

**터미널 2 - 프론트엔드 실행:**
```bash
cd frontend
npm run dev
```
→ "Local: http://localhost:5173" 같은 주소가 뜨면 성공!

### 4단계: 확인하기

브라우저에서 `http://localhost:5173` 접속
→ 로그인 화면이 보이면 성공! 🎉

## 🔧 파일별 역할 - 어디를 수정할까?

### 프론트엔드 (화면)

| 파일 | 역할 | 언제 수정할까? |
|------|------|----------------|
| `frontend/src/pages/Login.jsx` | 로그인 페이지 | 로그인 화면을 바꾸고 싶을 때 |
| `frontend/src/pages/PostList.jsx` | 게시물 목록 | 게시물 목록 디자인 변경 |
| `frontend/src/components/Header.jsx` | 상단 메뉴 | 상단바 수정 |
| `frontend/src/App.css` | 전체 디자인 | 색상, 폰트 등 |

### 백엔드 (기능)

| 파일 | 역할 | 언제 수정할까? |
|------|------|----------------|
| `backend/src/routes/auth.js` | 로그인/회원가입 API | 인증 로직 변경 |
| `backend/src/routes/posts.js` | 게시물 관련 API | 게시물 기능 추가/수정 |
| `backend/src/models/Post.js` | 게시물 데이터 구조 | 게시물에 필드 추가 |
| `backend/src/models/User.js` | 사용자 데이터 구조 | 사용자 정보 필드 추가 |

## 💡 꼭 알아야 할 용어 5개

1. **npm** = Node Package Manager, 코드 패키지를 설치하는 도구
2. **localhost** = 내 컴퓨터를 가리키는 주소
3. **포트(port)** = 컴퓨터의 문 번호 (3000번, 5173번 같은 거)
4. **API** = 프론트엔드와 백엔드가 대화하는 방법
5. **터미널** = 명령어를 입력하는 검은 화면

## ⚠️ 자주 나는 오류

### "포트가 이미 사용 중입니다"
→ 이미 프로그램이 실행되고 있다는 뜻
→ 해결: 기존 프로그램 종료 (Ctrl + C)

### "MongoDB 연결 실패"
→ MongoDB가 안 켜져 있다는 뜻
→ 해결: MongoDB 실행 또는 Docker 실행

### "모듈을 찾을 수 없습니다"
→ npm install을 안 했다는 뜻
→ 해결: `npm install` 다시 실행
