# 바이브(Vibe) 프로젝트 개요

## 🎯 바이브 프로젝트란?

바이브는 사용자가 글을 작성하고 공유할 수 있는 **소셜 미디어 플랫폼**입니다. 로그인, 게시물 작성, 수정, 삭제 등의 기본적인 기능을 가진 풀스택 웹 애플리케이션입니다.

## 🏗️ 프로젝트 구조

```
vibe/
├── frontend/          # 프론트엔드 (사용자가 보는 화면)
│   ├── src/
│   │   ├── pages/     # 각 페이지들 (로그인, 게시물 목록 등)
│   │   ├── components/# 재사용 가능한 UI 조각들
│   │   └── services/  # API 통신 코드
│   └── package.json
│
├── backend/           # 백엔드 (서버, 데이터 처리)
│   ├── src/
│   │   ├── routes/    # API 엔드포인트 정의
│   │   ├── models/    # 데이터베이스 구조 정의
│   │   └── middleware/# 중간 처리 로직
│   └── package.json
│
└── 강의자료/          # 학습 자료
```

## 🔧 사용 기술 스택

### 프론트엔드 (Frontend)
- **React**: 사용자 인터페이스를 만드는 JavaScript 라이브러리
- **Vite**: 빠른 개발 환경을 제공하는 빌드 도구
- **React Router**: 페이지 간 이동을 처리
- **Axios**: 서버와 통신하기 위한 HTTP 클라이언트

### 백엔드 (Backend)
- **Node.js**: 서버에서 JavaScript를 실행할 수 있게 해주는 환경
- **Express.js**: 웹 서버를 쉽게 만들 수 있는 프레임워크
- **MongoDB**: 데이터를 저장하는 NoSQL 데이터베이스
- **Mongoose**: MongoDB를 쉽게 사용하기 위한 라이브러리

### 기타
- **Docker**: 애플리케이션을 컨테이너로 패키징
- **Express Session**: 사용자 로그인 상태 유지

## 💡 핵심 개념

### 1. 클라이언트-서버 아키텍처

```
[사용자 브라우저] <----> [프론트엔드] <----> [백엔드] <----> [데이터베이스]
    (React)              (Express)      (MongoDB)
```

- **프론트엔드**: 사용자가 보고 클릭하는 화면
- **백엔드**: 데이터를 처리하고 저장하는 서버
- **데이터베이스**: 실제 데이터가 저장되는 곳

### 2. REST API

프론트엔드와 백엔드가 소통하는 방법입니다.

```
GET    /api/posts     → 게시물 목록 가져오기
POST   /api/posts     → 새 게시물 작성
PUT    /api/posts/:id → 게시물 수정
DELETE /api/posts/:id → 게시물 삭제
```

### 3. 인증 (Authentication)

사용자가 "로그인"했는지 확인하는 시스템입니다.
- 세션: 서버가 "이 사용자는 로그인했어"라고 기억
- 쿠키: 브라우저가 "나 로그인한 사용자야"라고 증명

## 🚀 주요 기능

### 1. 회원가입 / 로그인
- 사용자가 계정을 만들고 로그인할 수 있습니다
- 세션을 통해 로그인 상태를 유지합니다

### 2. 게시물 관리
- 로그인한 사용자는 글을 작성할 수 있습니다
- 자신의 글을 수정하거나 삭제할 수 있습니다
- 모든 사용자의 글 목록을 볼 수 있습니다

### 3. 권한 관리
- 자신의 글만 수정/삭제할 수 있습니다
- 로그인하지 않으면 글을 볼 수 없습니다

## 📊 데이터 흐름 예시

**게시물 작성 과정:**

1. 사용자가 글 작성 폼에 제목과 내용을 입력
2. "작성" 버튼 클릭
3. 프론트엔드가 `POST /api/posts` API 호출
4. 백엔드가 요청을 받아 검증
5. MongoDB에 데이터 저장
6. 백엔드가 "성공" 응답
7. 프론트엔드가 게시물 목록 페이지로 이동

## 🎓 학습 순서

1. **프론트엔드 기초**: React로 화면 만들기
2. **백엔드 기초**: Express로 서버 만들기
3. **데이터베이스**: MongoDB로 데이터 저장하기
4. **인증**: 로그인 시스템 이해하기
5. **통합**: 프론트엔드와 백엔드 연결하기
6. **배포**: 실제 서비스로 출시하기

## 🤔 왜 이렇게 나눠져 있을까?

### 관심사의 분리 (Separation of Concerns)

- **프론트엔드**: "어떻게 보여줄까?"에 집중
- **백엔드**: "데이터를 어떻게 처리할까?"에 집중
- **데이터베이스**: "데이터를 어떻게 저장할까?"에 집중

각 부분이 자기 역할만 하면 되니까:
- 코드가 깔끔해집니다
- 문제가 생기면 어디를 고쳐야 할지 명확합니다
- 여러 사람이 동시에 작업하기 쉽습니다
- 나중에 기능을 추가하기 쉽습니다
